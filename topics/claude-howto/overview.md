# Claude How To

## What It Is

Claude How To is a community-maintained, visual guide for learning Claude Code from beginner usage through more advanced patterns such as slash commands, memory, skills, subagents, MCP, hooks, plugins, checkpoints, and CLI workflows.

The repository is positioned as a practical companion to official documentation: less a specification, more a curated learning path with copy-paste examples, templates, diagrams, and install snippets that users can adapt into their own `.claude/` setups. The official Claude Code docs remain the source of truth for current behavior and compatibility.

## Key Concepts

- **Progressive learning path:** Organized into a roadmap spanning beginner through advanced usage, with an estimated 11-13 hour full path.
- **Copy-paste templates:** Includes example commands, CLAUDE.md patterns, skills, subagents, hooks, and MCP setup snippets intended for direct reuse.
- **Visual teaching style:** README emphasizes diagrams and example-driven onboarding instead of terse feature reference material.
- **Repository-as-catalog:** The project exposes concrete folders for `01-slash-commands` through `10-cli`, plus `CATALOG.md`, `INDEX.md`, and roadmap/reference docs.
- **Cross-language documentation:** English, Vietnamese, and Chinese entry points are visible from the repo homepage.

## Context

- **Who uses this:** Claude Code users who want practical examples faster than reading scattered official docs
- **When:** Best at onboarding, self-study, and browsing working patterns for `.claude/` customization
- **Why:** Reduces “I know the feature exists but not how to combine it” friction
- **How to use it safely:** Learn from it, mine patterns from it, but verify copied examples against official docs and local standards before adoption
